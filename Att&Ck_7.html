靶场 链接： http://vulnstack.qiyuanxuetang.net/vuln/detail/9/

红日靶场 att&ck 7

环境

  • 攻击机 kali： 192.168.9.106
  • 靶机 ubuntu：192.168.9.105

信息收集1

nmap

首先是用 fscan 扫描一下端口

fscan 真不错 GitHub: https://github.com/shadow1ng/fscan

看到扫描出来 redis(6379) 有未授权，可以写 /root/.ssh 或者 /var/spool/cron

redis 未授权

参照 https://www.cnblogs.com/loopkep/p/11135949.html， 写入 ssh 密钥

ssh 免密登录

信息收集2

ifconfig

看到目标有内网 192.168.52.0/24

fscan

既然有内网，就做一下内网的信息收集 fscan -h 192.168.52.0/24

得到两台目标机器

  • 192.168.52.30
    • windows 7， 可能存在 ms17-010
    • 8081: 通达 oa
  • 192.168.52.20

Socks 代理

这里用 nps 搭建代理

端口转发

把目标的 9999 端口转发到本地的 9999 端口

同理转发 40001， 40002 端口

Ms17_010

proxychains msfconsole，按照下图设置

getshell

之后的 192.168.93.20 也可以用 ms17_010 上线

cs 生成exe，上传执行，上线cs(40001端口)

信息收集3

ipconfig

发现目标还有 192.168.93.0/24 段

并且存在域环境

hashdump && mimikatz

获取用户hash

发现有域管(Administrator@WHOAMIANONY.ORG)的hash

定位域控

pth

有域管密码，直接pth 过去

成功上线

laravel debug mode (CVE-2021-3129)

192.168.9.105:81端口运行了 Laravel，底下有版本号 v8.29.0

网上搜索一下CVE，发现存在CVE-2021-3129，需要目标开启 debug mode

git找一下poc，https://github.com/SecPros-Team/laravel-CVE-2021-3129-EXP

直接利用，写入 fuckyou.php，用冰蝎2.x连接（根据poc的push时间找的版本），然后上传冰蝎3.0的 webshell

发现目标是一个docker

提权

发现 /home/jobs/shell 具有 setuid 权限

恰好有源代码

可以进行path劫持

得到 docker 的 root 权限

docker逃逸

参考文章：https://www.cnblogs.com/xiaozi/p/13423853.html

把宿主机的 根目录 挂载到 docker 的 /test 目录

用 echo 写入 /etc/crontab

写错了怎么办？ 用 sed -i '$d' crontab 可以删除最后一行数据

这里有点注意，如果直接在 * * * * * root 后面直接写反弹shell的命令会报错（原因暂时未知），所以把命令写进文件里，执行文件

监听端口，得到宿主机 root 权限
