fastjson

介绍

fastjson是阿里巴巴的开源JSON解析库，它可以解析JSON格式的字符串，支持将Java Bean序列化为JSON字符串，也可以从JSON字符串反序列化到JavaBean

题外话：之前看到有篇微博说有三个东西养活了大部分的安全人员，第三个就说是fastjson

小栗子5——初代 poc TemplatesImpl

设置成员变量

在 JSONObject.parseObject() 方法中，先设置了所有的 Feature 属性。

而后把 feature 传入到 parser 中连同 text 一起解析

在这个新的 parser.parse() 中最终会运行到 return deserializer.deserialze(this, clazz, fieldName)(DefaultJSONParser.parseObject: 368) 进行反序列化操作， 这里得到的 deserializer 是一个 JavaBeanDeserializer 对象。

在 JavaBeanDeserializer.deserialze() 方法中解析成员变量（设置参数）。

而在 JavaBeanDeserializer.parseField() 函数中，判断了 Feature 属性。

可以看到，在 JavaBeanDeserializer.parseField() 方法中，先判断了 Feature.SupportNonPublicField 参数。在这个参数被使用的情况下，找到 clazz 的所有成员变量，并设执行了 setAccessible(true) 然后作为他们（成员变量）的反序列化器，为之后的反序列化作准备。

在 fieldDeserializer.parseField() 方法中执行了熟悉的 setVal 操作

而这个 setVal ，则是通过 Field 来设置的

由此设置了成员变量。

触发漏洞

在轮到对 _outputProperties 字段执行 setValue() 操作的时候，会执行 getOutputProperties() 函数。

此时的 method 为 public synchronized java.util.Properties com.sun.org.apache.xalan.internal.xsltc.trax.TemplatesImpl.getOutputProperties()

在这个函数中执行的 newTransformer() 成了问题所在。

在 new TransformerImpl() 触发了漏洞。

这里执行了 newInstance() 操作，调用了构造器，而构造器是能够被控制执行任意代码的，由此产生了任意代码执行漏洞。
